GLB → 3D Tiles 変換ツール

GLBファイルをブラウザ上でメッシュ分割し、地理参照付き tileset.json と chunk GLB を ZIP で出力します。
位置・向きはすべて手動調整値です。自動推定は行いません。

このツールについて

GLB ファイルをブラウザ上で三角形単位に分割し、地理参照付き tileset.json と chunk GLB を ZIP で出力する Web ツールです。
サーバー不要・GitHub Pages でそのまま動作します。
出力は 3D Tiles 1.1 形式（GLB 直参照） です。B3DM ラッパーは不要です。

基本的な使い方

  • ① GLB を選択 — ドロップゾーンにファイルをドラッグ、またはクリックして選択
  • ② 分割チャンク数を設定 — 三角形数を均等に N 分割します。1 メッシュでも分割できます
  • ③ 地図で配置位置を確認・調整 — マーカーをドラッグするか、経度・緯度を直接入力
  • ④ 高さ・向きを入力 — 高さ（m）、Heading / Pitch / Roll を手動入力
  • ⑤「変換 & ZIP ダウンロード」を押す — ZIP が自動ダウンロードされます

出力ファイル構成

(GLBファイル名)_3dtiles.zip
  tiles_src/
    chunk_000.glb   ← 分割された GLB チャンク
    chunk_001.glb
    chunk_002.glb
    ...
  tileset.json      ← 3D Tiles エントリーポイント
  build_report.json ← 変換パラメータの記録

tileset.json の root.transform に ECEF 変換行列が入ります。 CesiumJS・Cesium ion・plateau-terria などモダンな 3D Tiles ビューアで読み込めます。

設定項目の説明

  • 分割チャンク数: 三角形数を均等に何分割するか。ファイルが大きい場合は 3〜10 程度を目安に。
  • 幾何誤差 (geometricError): ビューアがこのタイルを表示するかどうかの距離閾値（メートル）。建物スケールは 50〜200、都市スケールは 500 以上が目安。通常はデフォルト（200）のままで問題ありません。
  • 精緻化モード (refine): ADD＝子タイルを親タイルに重ねて追加表示、REPLACE＝子タイルで親タイルを差し替え。
  • 経度・緯度: モデルを配置する地点の座標。地図のマーカーをドラッグしても変更できます。
  • 高さ (m): 地面からの高さ。建物の場合は 0 か地面の標高を入力。
  • Heading: 北を 0° として時計回りの回転角。モデルの正面方向を調整します。
  • Pitch: 前後の傾き。通常は 0。
  • Roll: 左右の傾き。通常は 0。

注意事項

  • 位置・向きはすべて 手動入力値 です。自動推定は行いません。
  • 大きな GLB（目安：100MB 超）はブラウザのメモリを圧迫することがあります。
  • B3DM 変換はこのツールでは行いません。必要な場合は ZIP 解凍後に npx 3d-tiles-tools glbToB3dm を別途実行してください。
  • テクスチャ付き GLB の場合、マルチマテリアルが単一マテリアルに統合されることがあります。

用語集

  • 3D Tiles: 大規模 3D 地理空間データの配信・表示のためのオープン規格（OGC標準）
  • tileset.json: 3D Tiles のエントリーポイントファイル。タイル構造・変換行列・ファイル参照を記述します
  • チャンク (chunk): 元の GLB を分割した部分ファイル（chunk_000.glb など）
  • root.transform: モデルのローカル座標系を地球上の位置（ECEF）に変換する 4×4 行列
  • ECEF: 地球中心・地球固定座標系。3D Tiles の transform はこの座標で指定します
  • ENU: East-North-Up。配置地点を基準とした局所座標系。ECEF との橋渡しに使います
  • Heading / Pitch / Roll: モデルの向きを表す 3 軸の回転角。それぞれ Z・Y・X 軸回転に対応します
